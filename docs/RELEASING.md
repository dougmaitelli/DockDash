# Release process

DockDash uses semantic version tags and GitHub Actions to publish releases and container images.

## Versioning

Release tags use the form `vMAJOR.MINOR.PATCH`:

- **MAJOR** for incompatible configuration, API, persistence, or behavior changes
- **MINOR** for backward-compatible functionality
- **PATCH** for backward-compatible fixes

Pre-release tags may be used when explicitly coordinated, but the current image-tag workflow is optimized for stable semantic versions.

## Continuous delivery

The workflow in `.github/workflows/ci.yml` performs the following:

- Pull requests to `master`: lint, typecheck, and tests
- Pushes to `master`: validation and publication of the `dev` container image
- Pushes of `v*` tags: validation, GitHub release creation, and publication of `latest` plus semantic-version image tags

Release notes are generated by git-cliff using `cliff.toml`. Clear commit subjects produce clearer changelogs.

## Preparing a release

1. Ensure `master` is current and CI is green.
2. Review changes since the previous tag, including configuration, migrations, and security impact.
3. Update documentation and `.env.example` for any new settings.
4. Run the project checks locally:

   ```bash
   yarn typecheck
   yarn lint
   yarn test
   yarn build
   ```

5. Choose the next semantic version.
6. Create and push an annotated tag:

   ```bash
   git tag -a v1.2.3 -m "DockDash v1.2.3"
   git push origin v1.2.3
   ```

7. Verify the GitHub release and GHCR image tags after the workflow completes.
8. Smoke-test the published image with a fresh database and, when migrations changed, a copy of an existing database.

## Correcting a release

Do not move or overwrite a published version tag. Fix the issue on `master` and publish a new patch version. If a release is unsafe, mark it accordingly in GitHub and document the affected versions in the new release notes or security advisory.

## Rollback

Deployments should pin a known version rather than relying exclusively on `latest`. To roll back, redeploy the previous image tag. Database migrations may make application rollback unsafe; back up the SQLite database before upgrading and review migration compatibility first.
